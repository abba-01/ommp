
```markdown
### 🟡 Medium (P2)
```

## 🚀 Session Initialization Template

```markdown
=== SESSION START ===
Date: [YYYY-MM-DD HH:MM]
Session ID: [S###]

CONTEXT_INJECTION_START
↳ Current [PROJECT_NAME]_tracker (last [N] items):
   [Paste relevant section]
↳ Session objective: [Specific goal for this session]
↳ Constraints: [Any session-specific limitations]
CONTEXT_INJECTION_END

Task: [Specific instruction for AI]
Output format: [bullets/code/prose/artifact]
```

## 🏷️ Task Status System

Choose your tracking style:

### Option A: Simple Status
- `[ ]` = Open
- `[✓]` = Complete
- `[→]` = In Progress
- `[?]` = Blocked/Need Info

### Option B: Detailed Status
- `[✔]` Done → [completion reference]
- `[🔄]` Active → [current status]
- `[⏸️]` Parked → [reason/dependency]
- `[❓]` Blocked → [what's needed]
- `[🚫]` Cancelled → [reason]

## 🔄 Work Cadence Framework

### Sprint Pattern (customizable)
```markdown
1. START (5 min)
   - AI echoes understanding of goals
   - Reviews open items (max [N] lines)
   - Proposes work order

2. EXECUTE (main work)
   - Deliver [deliverable type]
   - Format: [preferred format]
   - Scope: [scope limits]

3. CLOSE (5 min)
   - Update task statuses
   - Delta summary: "Completed: X, Active: Y, New: Z"
   - Next session prep notes
```

## 📊 Output Templates

### For Code Projects
```markdown
Request: "Provide [patch/diff/full file] format only"
- Use diff --git format for changes
- Include line numbers for context
- Minimal prose unless explaining complexity
```

### For Writing Projects
```markdown
Request: "Deliver in [format]: 
- Outline only
- Full draft with headers
- Bullet summaries
- Annotated revision
```

### For Analysis Projects
```markdown
Request: "Structure as:
- Executive summary (3 bullets)
- Data/findings (tables/charts)
- Recommendations (numbered)
- Next steps (checklist)
```

### Token Management

## 🚦 Quick Start Commands

Copy and customize these starter prompts:

### New Session Start
```
CONTEXT: [PROJECT_NAME] tracker attached
Goal: Complete next [N] priority items
Format: [preferred output format]
Constraints: [time/scope/technical]
BEGIN with status check of items [X-Y]
```

### Mid-Session Checkpoint
```
STATUS CHECK: Summarize progress on current tasks
NEXT: Propose order for remaining items
TIME: [X] minutes remaining
```

### Session Close
```
WRAP UP:
1. Update all task statuses
2. Create delta summary
3. Note any blockers
4. Prep notes for next session
```
